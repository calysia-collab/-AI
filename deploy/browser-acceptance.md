# 瀏覽器與手機正式驗收清單

只有在 staging 使用有效 HTTPS 憑證、正式 PostgreSQL 與 ClamAV 時，才能簽署本清單。

## 測試裝置

- Windows：最新版 Chrome、Edge。
- macOS：最新版 Safari、Chrome。
- iPhone：目前受支援的 iOS Safari。
- Android：目前受支援的 Chrome。
- 手機至少測試一台實機，不可以只用桌面縮放代替。

## 必測流程

1. 開啟登入頁，確認沒有憑證警告或混合內容警告。
2. 登入張經理帳號，重新整理後仍維持登入。
3. 登出後按上一頁，確認無法重新看到客戶資料。
4. 新增、修改、搜尋及刪除測試客戶。
5. 新增保單，確認保單與客戶關聯正確。
6. 拍攝或選擇 JPEG、PNG、WebP 保單圖片。
7. 確認乾淨附件可下載，下載網址逾期後失效。
8. 確認不支援格式、超過 10 MB 及惡意測試檔被拒絕或隔離。
9. 新增行事曆事件，完成檢閱後通知標記消失。
10. 檢閱情報周報，確認通知狀態正確。
11. 測試團隊帳號的 owner、manager、advisor、viewer 權限。
12. 進入客戶系統、商品比較、團隊管理及每日語錄頁面。
13. 關閉網路後嘗試新增客戶，確認系統阻止送出且不在瀏覽器明文保存。
14. 恢復網路後重新登入，確認伺服器資料沒有重複或遺失。
15. 將網站加入手機主畫面，確認 PWA 圖示、啟動畫面及返回操作正常。
16. 開啟客戶 360，新增、修改及封存聯絡方式、關係人、互動、任務、文件與同意紀錄。
17. 上傳 CSV／XLSX 客戶名單，確認進度、取消及錯誤檔下載。
18. 以客戶姓名、電話或保單資料搜尋，確認只看到有權限的資料。
19. 完成 OCR 辨識、低信心欄位修正及核准建檔，確認未核准前不會建立保單。

## 安全檢查

- Session Cookie 必須包含 `HttpOnly`、`Secure`、`SameSite=Strict`。
- `/api/state`、`/api/metrics`、`/api/audit` 未登入時必須回傳 `401`。
- 修改 API 缺少 CSRF token 時必須回傳 `403`。
- 瀏覽器儲存空間不得出現客戶、保單、行程或團隊明文。
- 伺服器回應必須包含 CSP、HSTS、`X-Content-Type-Options`。
