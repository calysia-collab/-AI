- 新增 CSRF 防偽請求驗證，保護所有新增、修改與刪除 API。
- 密碼連續錯誤 5 次後暫時鎖定 15 分鐘。
- 新增 `owner`、`manager`、`advisor`、`viewer` 角色基礎；稽核與監控限主人或管理者。
- 未登入時客戶、保單、行程、同步、稽核與監控 API 一律拒絕存取。
- 新增安全登入頁、首次設定頁、登出按鈕與登出後敏感瀏覽器快取清除。
- PostgreSQL 正式版結構已補上組織、使用者與登入工作階段資料表。
- 主人可在團隊管理頁新增登入帳號、調整角色、停用／啟用帳號與重設密碼。
- 客戶、保單、行程、團隊、任務、設定、版本號及稽核紀錄均依 `organization_id` 隔離。
- 舊版未歸戶資料會在首次設定或單一組織升級時自動歸入原工作台。
- 帳號停用、角色調整或密碼重設時，既有登入工作階段立即失效。
- 顧問不可使用整批同步 API 修改團隊資料；唯讀角色不可進行資料異動。
- SQLite 以觸發器阻擋跨組織保單與行程關聯。
- 已用實際 Edge 完成首次設定、登入、團隊頁與建立帳號互動驗證。
- 使用者可自行變更密碼；變更後所有舊登入工作階段立即失效。
- 新增驗證器 App TOTP MFA、加密秘密、驗證碼重播防護與 10 組一次性復原碼。
- MFA 復原碼可下載保存，離開設定視窗或完成啟用後會從畫面記憶體清除。
- 新增本機主金鑰建立與讀取；正式環境可由 `SASHA_MASTER_KEY` 注入 32 位元組金鑰。
- 錯誤 JSON 與過大請求會回傳受控錯誤，不再誤報為伺服器內部故障。
- 自動測試增至 62 項，全部通過。
- 後端登入與 API 已改為同步／非同步資料庫皆可使用，SQLite 操作維持相容。
- 新增 PostgreSQL 連線池、完整帳號／MFA／客戶／保單／行程／團隊／稽核資料層。
- 新增啟動前資料表遷移、跨程序鎖定、遷移校驗碼及重複執行保護。
- 新增 `verify-full` TLS、連線池大小與資料庫引擎健康狀態。
- 新增 SQLite 一致性快照轉移；只允許空白 PostgreSQL，並逐表驗證筆數。
- 轉移時不搬移既有登入工作階段，所有人須在新環境重新登入。
- 客戶新增穩定的 `owner_user_id` 負責人帳號識別；顧問改名時客戶顯示名稱會同步更新。
- 主人與管理者可指派客戶，顧問新增客戶時由後端強制指派本人，不能竄改成其他顧問。
- 顧問只能讀寫自己負責的客戶、相關保單及客戶行程；未綁定客戶的團隊行程維持團隊共用。
- SQLite 與 PostgreSQL 均在資料庫查詢層套用相同的顧問資料範圍。
- 舊客戶只有在顧問姓名可唯一對應同組織有效帳號時才自動補上負責人識別，避免錯誤歸戶。
- 新增僅能由主機維護人員執行的緊急帳號復原工具，不開放匿名密碼重設 API。
- 緊急復原會驗證新密碼強度、拒絕沿用舊密碼、清除帳號鎖定、使既有登入失效並留下稽核紀錄。
- 若驗證器與 MFA 復原碼同時遺失，可用明確參數停用 MFA 並清除舊復原碼。
