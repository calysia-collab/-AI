# 莎莎保險助理工作台

## 安全登入

1. 第一次執行 `start.bat` 後，畫面會要求建立張經理的主人帳號。
2. 密碼至少 12 個字元，並包含英文大寫、英文小寫與數字。
3. 登入工作階段最長 12 小時，閒置 30 分鐘後需重新登入。
4. 密碼連續錯誤 5 次，帳號會暫時鎖定 15 分鐘。
5. 客戶、保單、行程與團隊資料不會寫入瀏覽器 `localStorage`；登出時會再清除舊版殘留資料。
6. 登入後可從右下角個人設定變更自己的密碼，舊登入工作階段會立即失效。
7. 可啟用驗證器 App 的 MFA；10 組復原碼皆只能使用一次，離開設定視窗後不再顯示。

所有客戶、保單、行事曆、稽核與監控 API 都已要求登入；新增、修改與刪除資料另有 CSRF 防偽驗證。正式對外部署仍必須使用 HTTPS、反向代理與受管理的 PostgreSQL。

客戶現在以登入帳號識別負責顧問。主人與管理者可指派客戶；顧問只會讀寫自己負責的客戶、相關保單與客戶行程，並可查看未綁定客戶的團隊行程；唯讀成員可查看組織資料但不能異動。

這是一個保險業務經理使用的安全開發版網站，包含：

- 智慧行事曆、提醒與快速建立行程
- 保單圖片上傳、辨識流程、欄位校正與客戶檔案保存
- 國際財經、台灣財經保險及法規情報週報
- 保險商品篩選與並排比較
- 每日早上 8 點更新的古典書籍語錄、原文、篇章出處與 AI 解讀
- 團隊成員、共同目標、進度與協作任務管理
- 客戶名單、服務階段、保障需求與追蹤日期管理
- 各保險公司最新消息、建議書、要保書及商品文件入口

## 開啟方式

電腦需使用 Node.js 24.14.x，再雙擊 `start.bat`。

也可以在專案資料夾執行：

```text
npm start
```

客戶、保單、行程、團隊與帳號資料以伺服器資料庫為唯一正式來源。連線中斷時系統會阻止敏感資料提交，不會把資料明文保存在瀏覽器。

本機第一次啟動時會在 `.data/master.key` 建立加密金鑰。MFA、客戶、保單、行程、附件與備份均使用加密服務；備份與金鑰必須分開保管。正式環境必須由秘密管理或雲端 KMS 注入 `SASHA_MASTER_KEY`、`SASHA_DATA_KEY_ID` 與 `SASHA_DATA_KEYS`。

正式環境可設定 `SASHA_DATABASE_URL` 切換為 PostgreSQL。啟動前會自動執行帶鎖定與 checksum 的不可變資料庫移轉；未設定時仍使用本機 SQLite。部署使用 `npm ci`，不得用 `npm install` 任意更新鎖定版本。

## 階段 3

- 已新增 PostgreSQL `006` OCR 工作、辨識欄位、人工修正與稽核結構。
- 保單圖片必須通過附件安全掃描後才能建立 OCR 工作。
- OCR 結果會顯示欄位信心值，必須由使用者檢閱、修正及明確核准後才建立保單。
- OCR 供應商採可替換介面；測試環境使用固定測試資料，正式環境必須設定 HTTPS OCR 服務。
- 未完成 OCR 工作會在服務重啟後自動續跑。
- 完成報告請參閱 `階段3建置完成報告-2026-06-13.md`。

目前後端只允許本機 `127.0.0.1` 直接存取。正式遠端部署應讓 HTTPS 反向代理連入本機服務，不應直接開放 Node.js 連接埠。

執行完整檢查：

```text
npm run verify
```

## 維護與備份

- 健康狀態：`/api/health`
- 營運指標：`/api/metrics`
- 操作稽核：`/api/audit`
- 建立加密備份：`node scripts/backup.mjs`
- 驗證備份：`node scripts/verify-backup.mjs 備份檔`
- 還原預覽：`node scripts/restore-backup.mjs --source 備份檔`
- PostgreSQL 結構遷移：`npm run postgres:migrate`
- SQLite 轉移預覽：`npm run postgres:import-sqlite -- --source=備份檔`
- 帳號復原預覽：`npm run account:recover -- --username=登入帳號`
- PostgreSQL 實機整合測試：`npm run postgres:test`
- 移轉檔鎖定驗證：`npm run migrations:verify`
- 階段 2 結構與筆數核對：`npm run phase2:migration:verify`
- 階段 2 十萬筆分頁壓力測試：`npm run phase2:scale`
- PostgreSQL 十萬筆分頁與盲索引測試：`npm run phase2:postgres-scale`
- 正式版完整檢查、備份還原及金鑰輪替：`npm run verify:release`

完整流程請參閱 `營運維護手冊.md`。
